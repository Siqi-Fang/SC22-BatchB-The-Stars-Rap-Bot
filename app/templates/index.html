Sentiment Analysis

The purpose of the sentiment analysis is to give the Lyrics generation a prompt to work with, generated from user's input. However that input will be subconsciously derived from the user's emotions using sentiment analysis. NLTK and text2emotion were both used to check each word of the user's input and define an emotion based on the words. The results are outputted as an emotion.

Generating Lyrics

Lyrics are generated via a custom Lyrics dataset utilizing the Genius API, and cleaned with custom parameters for unwanted words that appear in the file such as [Verse] as well as parameters such as [Embed]. We then use AI text gen to produce lyrics based on the custom dataset. Following that, we use Datamuse then to create rhymes of the end of sentences and using the Roberta neural network to generate words in between the original sentence and the rhymed last words to make the sentence make sense. Post processing, we removed swear words, as well as select 4 of the best lines that fit around 12 syllables.

Speech generation

After getting input from the Lyrics generator, we turned the lyrics into speech using Amazon Polly. Using SSML tags, we will be able to select from a variety of AI voices and influence the speed in which the AI raps the lyrics. Due to time and current technology constraints, we were not able to make the AI rap in different "flows", due to unstable syllable counting APIs currently existing. Thus we had to default to the AI rapping the lyrics relatively to the beat of the music.
